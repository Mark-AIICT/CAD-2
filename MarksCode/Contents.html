01-Out-Parameter-WPF An OUT parameter is similar to a return value from a method but it's defined in the parameters section. You don't pass a value when calling the method, instead you look at the value after the method has run. In this example I'm calling the TryParse method which returns true or false, the OUT parameter will have a value after the TryParse has been called.
02-Out-Parameter In this example I've written my own function that includes the out specifier for one of the parameters.

04-Tracing-WPF This is an example of an application that writes 'Log' entries using a special mechanism that is part of the .net framework called 'trace listeners'. using this mechanism system operators can change the application configuration to increase or decrease the detail of the log entries. Additionally, they can log to other locations without having to change the application code.
05-PerformanceCounters (Multi-Instance) A multiple instance performance counter can be instatiated a number of times. This application creates a performance category group as well as a couple of performance counters.
06-PerformanceCounters (Single-Instance) A single instance performance counter is effectively shared across several applications. This application creates a performance category group as well as a couple of performance counters.
07-TraceSourceDemo Application showing the use of the TraceSource object. This object was introduced in .Net 2.0 and is a better choice than the TraceSwitch object you saw in an earlier demo.
